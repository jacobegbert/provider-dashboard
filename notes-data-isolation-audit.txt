DATA ISOLATION AUDIT - Procedures that accept patientId without ownership check

The approach: For patient-facing procedures, we need to verify that ctx.user is either:
1. The admin/provider (role === "admin") — they can access any patient
2. The patient themselves (their userId matches the patient's userId)

We'll create a helper: ensurePatientAccess(ctx, patientId) that:
- If ctx.user.role === "admin", allows access (provider can see all)
- Otherwise, looks up the patient and checks patient.userId === ctx.user.id
- Throws FORBIDDEN if neither condition is met
